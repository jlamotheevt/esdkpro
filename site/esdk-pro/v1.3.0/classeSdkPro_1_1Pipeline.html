Detailed Description

All data processing in eSDK Pro is managed through the Pipeline. The Pipeline is a directed graph of tasks, each task implements its own data processing to convert input data into output data.

To process data from a camera: 1: A camera task must be created. 2: The desired data processing task must be created. 3: The camera task output port must be connected to the input port of the data processing task.

Eg. For 2 cameras to do some processing, the pipeline graph must look like the following. [camera task 1] -> [process task 1] [camera task 2] -> [process task 2]

To execute the pipeline, Pipeline::Start() must be called which first initializes all servers with the pipeline graph, then starts all cameras within the pipeline. The pipeline will continue executing until Pipeline::Stop() is called or a task worker calls TaskWorker::Abort().

No changes to the pipeline graph while the pipeline is running is permitted.

Creates a plugin task for executing custom code within a pipeline. Plugins are created by implementing a TaskWorker and registering the TaskWorker with eSDK Pro.

  • Local Server: Call the RegisterTaskPlugin function to register the TaskWorker. See plugin_static example.
